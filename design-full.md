# Design System

## Overview
A comprehensive 16:9 video explaining the NVIDIA Claude Code Proxy project, starting with the problem statement and walking through the complete solution.

## Colors
- **Background**: `#0a0a0a` (deep black)
- **Foreground**: `#e0e0e0` (light grey for text)
- **NVIDIA Green**: `#76b900` (primary accent)
- **Anthropic Orange**: `#d97757` (secondary accent)
- **Surface**: `#1a1a1a` (card backgrounds)
- **Border**: `#333333` (subtle borders)
- **Code Background**: `#0d0d0d` (dark code blocks)

## Typography
- **Headlines**: Inter, 700 weight, 48-64px
- **Body**: Inter, 400 weight, 18-24px
- **Code**: JetBrains Mono, 400 weight, 14-18px
- **Labels**: Inter, 600 weight, 14-16px, uppercase

## Motion
- **Energy**: Moderate - smooth, professional
- **Easing**: `sine.inOut` for ambient, `expo.out` for entrances, `power2.in` for exits
- **Duration**: 0.6s entrances, 2-3s holds, 0.8s transitions
- **Atmosphere**: Radial glows, grid patterns, animated SVG paths

## Components
- **Cards**: 12px border-radius, subtle borders, dark surface
- **Code blocks**: 8px border-radius, monospace font, syntax highlighting
- **SVG Icons**: Animated paths, stroke-dasharray animations
- **Arrows**: Animated showing data flow
- **Badges**: Pill-shaped, accent backgrounds

## Do's and Don'ts
- DO use smooth, continuous motion
- DO show data flow with animated arrows
- DO highlight code snippets with syntax colors
- DO use SVG animations for technical concepts
- DON'T use harsh jump cuts
- DON'T overuse gradients
- DON'T make text smaller than 16px for body copy
- DO ensure all setup instructions match README exactly
